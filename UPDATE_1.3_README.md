# Bulletin Board Mod — Обновление 1.3 "Living World"

## 📋 Обзор обновления

Обновление **1.3 - Living World** добавляет динамическую генерацию записок на досках объявлений и интеграцию с генерацией мира.

---

## ✨ Новые возможности

### 1. 🎲 Система случайных записок

**RandomNotePool** — генератор случайных записок с предопределёнными шаблонами.

#### Категории записок:
| Категория | Описание | Цвет печати |
|-----------|----------|-------------|
| **ANNOUNCEMENT** | Объявления (пропажа, поиск работы, продажа) | Оранжевый |
| **WARNING** | Предупреждения (криперы, карантин, зомби) | Красный |
| **PERSONAL** | Личные сообщения (поздравления, поиски) | Голубой |
| **QUEST** | Квесты (заказы, награды) — задел на будущее | Зелёный |

#### Примеры записок:
- "Пропала курица!" — объявление
- "Осторожно, криперы!" — предупреждение
- "С днём рождения!" — личное
- "Нужен уголь" — квест

---

### 2. ⏰ Авто-размещение записок

**AutoNoteScheduler** — автоматически размещает случайные записки на досках каждые **15 секунд**.

#### Как работает:
1. Каждые 15 секунд планировщик ищет доски в загруженных чанках вокруг игроков
2. Выбирает случайную доску со свободными слотами
3. Размещает случайную записку из пула

#### Управление планировщиком:
```
/bulletin scheduler enable    # Включить авто-размещение
/bulletin scheduler disable   # Выключить авто-размещение
/bulletin scheduler trigger   # Принудительно разместить записку
/bulletin scheduler status    # Показать текущий статус
```

**По умолчанию:** ВКЛЮЧЕНО

### 3. 🎪 Ивенты скидок жителей

**VillageDiscountEvent** — создаёт временные скидки у жителей с информированием через доски объявлений.

#### Как работает:
1. Выбирается случайный житель с профессией
2. В радиусе 50 блоков находится доска объявлений
3. Создаётся записка **без печати** (исчезнет через 1 час)
4. Записка информирует о скидке у жителя
5. **Если установлен Trade Overhaul** — применяется **реальная скидка 50%** к ценам жителя

#### Особенности:
- **Длительность скидки:** 1 час (3600000 мс)
- **Размер скидки:** 50%
- **Записка без печати:** автоматически исчезнет, когда скидка закончится
- **Автор записки:** профессия жителя (например, "Фермер", "Кузнец")

#### 🔗 Интеграция с Trade Overhaul:
- **Если Trade Overhaul установлен:** Скидки **реально работают** — цены уменьшаются на 50%
- **Если Trade Overhaul НЕ установлен:** Создаются только **информационные записки**

> 💡 **Проверь статус:** Команда `/bulletin discount list` покажет `[Trade Overhaul ENABLED]` или `[Trade Overhaul DISABLED]`

#### Команды:
```
/bulletin discount trigger    # Создать ивент скидки
/bulletin discount list       # Показать активные скидки
/bulletin discount clear      # Снять все скидки
```

#### Пример записки:
```
Заголовок: СКИДКА!
Текст: У Джон (Фермер) скидка 50% на все товары!
       Успейте воспользоваться!
Автор: Фермер
```

#### Пример вывода команды list:
```
[Trade Overhaul ENABLED]
=== Активные скидки (2) ===
  REAL Джон (Фермер) - 55 мин. осталось
  REAL Стив (Оружейник) - 30 мин. осталось
```

### 4. ⚡ Команды для событий

Все команды требуют уровень прав **2** (операторы/администраторы).

#### `/bulletin trigger [radius]`
Создаёт случайную записку на ближайшей доске в радиусе.
```
/bulletin trigger        # радиус 10 блоков по умолчанию
/bulletin trigger 50     # радиус 50 блоков
```

#### `/bulletin trigger-at <x> <y> <z>`
Размещает случайную записку на указанной доске.
```
/bulletin trigger-at 100 64 200
```

#### `/bulletin clear <x> <y> <z>`
Очищает все записки с указанной доски.
```
/bulletin clear 100 64 200
```

#### `/bulletin list <x> <y> <z>`
Показывает список всех записок на доске.
```
/bulletin list 100 64 200
```

#### `/bulletin scheduler <enable|disable|trigger|status>`
Управление авто-размещением записок.
```
/bulletin scheduler enable    # Включить
/bulletin scheduler disable   # Выключить
/bulletin scheduler trigger   # Разместить прямо сейчас
/bulletin scheduler status    # Показать статус
```

## 🎮 Использование

### Для тестирования:
1. Поставь доску объявлений командой:
   ```
   /give @s bulletin-board:bulletin_board
   ```

2. Размести записку:
   ```
   /bulletin trigger-at ~ ~1 ~
   ```

3. Проверь записки:
   ```
   /bulletin list ~ ~1 ~
   ```

### Для интеграции с деревней:
1. Создай структуру (см. STRUCTURE_GUIDE.md)
2. Добавь NBT файл в проект
3. Протестируй генерацию:
   ```
   /locate structure bulletin-board:village_bulletin_board
   ```
